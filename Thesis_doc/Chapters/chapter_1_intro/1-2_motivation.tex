\subsection{Potentials of Distributed Source Coding in Federated/Distributed Learning}

Communication is the dominant bottleneck in federated learning. The data exchanged between workers and the server mainly consists of two parts:
\begin{enumerate}[nosep]
    \item Model parameters $\theta$, which are sent from the server to the workers for initialization.
    \item Gradients $g$, which are computed locally at the workers and transmitted back to the server for aggregation.
\end{enumerate}

It has been repeatedly demonstrated in the literature that gradients communicated by different nodes are strongly correlated.
This is expected, since these gradients collectively contribute to forming a global model with improved generalization capabilities, so we can assume that they are all generally trending towards the same optimal point.
Given this correlation, the advantages of applying Distributed Source Coding (DSC) to compress gradients become evident; DSC is specifically designed to exploit dependencies among correlated sources, making it a natural fit for distributed learning scenarios.
It's important to note that depending on how you group gradients, the correlation could be different which could be exploited in different ways; in our work, we focus on individual parameters, which is why it's called Wyner-Ziv Qauntizer.

Below we see empirical evidence gathered by other works, showing the link between gradients from different workers, and group either by layer or not group (using the entire gradient vector).

The figure \ref{fig:correlation - LGCD-DL - MI per layer}, shows the difference between marginal entropy and MI between 2 worker gradients. this discrepancy is the information unique to each worker, which is the part that cannot be compressed using DSC. We can see that this difference is relatively small, indicating a high potential for compression using DSC. it also indicates that the unique component is larger in the first and last layers, which is expected since the first and last layers are more effected by the data batch due to proximity to input and output layers.

\subsection{Learned Wyner-Ziv Quantizer in Federated Learning}
\subsubsection{Differences with past DSC implementations in Federated Learning}
The existing approaches to gradient compression vary in foucs, from compressing the entire model vector (e.g., low‑precision or entropy coding of the concatenated gradient) to per‑parameter compression, i.e, quantization, ranging from binary/ternary schemes to learned quantizers, with occasional DSC‑like ideas layered in.
Among the works on quantization leveraging DSC, there as been more foucs on classical DSC constructions, which are complex and resource‑intensive. Our approach differs by using a learned layerd (sucsessive) WZ quantizer trained on‑the‑fly. This offers two practical advantages: it is computationally lighter than conventional code‑design pipelines, and it is flexible, adapting to the statistics encountered during training.
