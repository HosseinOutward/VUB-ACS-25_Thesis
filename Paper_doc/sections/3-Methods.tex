We present our framework which exploits multi-terminal approach on correlated gradients. This correlation has been observed in prior works~\cite{ZhangTao2022, Abrahamyan2022LGC} and confirmed for per-coordinate level in~\ref{fig:grad_corr}. The distribution shift was investigated and shown to be small, assuming scales are normalized. These facts enable us to use DSC methods, like learned WZ quantizer from works such as~\cite{BorisLayeredWZ, OzywzRecoverBinning}.


\subsection{Proposed Protocol leveraging WZQ}
\label{subsec:method-protocols}
The proposed protocol is composed of a sequence of \emph{Round Types} and maximum rate for each round \(r\). This will determine how the multi-stage quantizer  \(Q_{k,r} = (E_{k,r}, D_{k,r})\) (based on ~\cite{BorisLayeredWZ,OzywzRecoverBinning}) is trained for each worker \(k\), by defining the training target \(T_{k,r}\), and where training occurs (server or worker), number of output symbols \(B_{r}\), number of stage \(J_{r}\) and lastly, the side information (\textbf{SI}).

The SI is denoted as \(S_{k,r,i} \triangleq \{\hat g_{q,p,i}\}_{(q,p)\in \mathcal{I}_{k,r}}\) where \(\hat g_{q,p,i}\) is the reconstructed gradient of worker \(q\) at round \(p\), and $\mathcal{I}_{k,r}$ is a round-type–dependent index set selecting a subset of previously available reconstructions (possibly from other workers) for coordinate $i$. The SI used for training of \(Q_{k,r}\) is also used for inference and is unchanged until the next round with training.

Now we will first describe the quantization process itself, and then the different round types and their sequence in the protocol timeline.

\subsection{Learned Successive Multi-terminal Wyner--Ziv Quantization (WZQ)}
\label{subsec:method-lsmtwzq}
At round \(r\), worker \(k\) holds flattened accumulated gradient \(G_{k,r}\in\mathbb{R}^n\), preprocessed to \(X_{k,r}\) by normalizing per layer.
Using a multi-level WZ quantizer encoder \(E_{k,r}\) with \(J\) stages, we quantize each element of \(X_{k,r}\) into a list of symbols \[\{u_{k,r,i,j}\}_{j}.\], with spatial index \(i\) indicating the parameter and \(j\) showing quantizer refinement stage.
Information on training said quantizer is in Sec.~\ref{subsec:method-protocols}.

For each refinement stage \(j\), we aggregate the symbols \(\{u_{k,r,i,j}\}_{i}\) across all parameters into \(U_{k,r,j}\), which can then be further compressed into bitstreams \(O_{k,r}\) using an additional lossless DSC method like SW coding. This will be transmitted to the server, and with the help of the SW decoder and the SI for all parameters, \(U_{k,r,j}\) can be recovered, and reorganized into \(\{u_{k,r,i,j}\}_{i}\) for each parameter.

Using the quantizer decoder \(D_{k,r}\) combined with SI, we reconstruct \(\hat X_{k,r}\), which is finally to be post-processed into \(\hat G_{k,r}\). Figure~\ref{fig:block} illustrates the process.

Note that for post-processing, we need the scale factors, one per layer, used in pre-processing, which can be transmitted with negligible cost. Also, since the quantizer is trained on one node, we also share the decoder parameters, which also has negligible cost compared to the quantization bitstream.


There are two distinct gains from our approach, one from \(G_{k,r}\) to \[\{u_{k,r,i,j}\}_{j}.\] and another when going to \(O_{k,r}\). We call them \emph{Binning gain} and \emph{Context Gain} respectively. These two gains will compete, i.e. improving one can reduce the other. Therefore, we need to apply both sequentially to get the full picture of the rate gains. But for our experiments, we do not use SW coding due to implementation complexity, and instead estimate the rate gains by computing the entropy of \(U_{k,r,j}\) conditioned on the SI (i.e. \(H(U_{k,r,j}|S_{k,r})\)). This is done by fitting a neural network to predict the distribution of \(U_{k,r,j}\) with only the SI as input. (note to author: also maybe use a conditional entropy coder like rANS to get a more accurate estimate of the lower bound).

\subsection{Training Quantizers and SI variants}
We consider two main round types, \emph{All-SI} and \emph{Temporal-only} and two additional round types, \emph{Pretrained} and \emph{Freeze} which are selected per round by the protocols round type sequence.

\textbf{All-SI round:}
Trained on the server, SI consists of all and any workers' past reconstructions except for one; the most recent reconstruction (for the worker in question) is excluded and used as the training target. This leverages both inter-node and temporal correlation but introduces a train--inference distribution mismatch, which we observe is usually the dominant cause of higher rate--distortion cost. Another issue is that the resulting higher distortions degrade the reconstructions used on the next training phases, leading to multi-round error accumulation---an issue we address with the next round type.
The SI used for computing the conditional prior for context gain estimation is the same as the SI plus the training target.

\textbf{Temporal-only round:}
Trained on the worker, with SI consisting of only the same worker's past reconstructions from other temporal-only rounds. Training target is the true gradient to be transmitted, minimizing train--inference mismatch. Because the decoder is available at both ends, the server and worker can maintain an identical list of reconstructions. The benefit of low train--inference mismatch comes at the cost of exploiting only temporal (not inter-node) correlation, so this round type is used periodically to realign reconstructions to the true gradients and mitigate error accumulation.

\textbf{Pretrained round and Freeze rounds:}
Pretrained round is the first round of the protocol, where a non-WZQ quantizer is trained on a Laplacian source without SI. Importantly, since it doesn't use SI, by also sending the decoders to the workers, we can use the resulting reconstructions as SI similar to the temporal-only round.

Freeze rounds are will use the most recently trained quantizer from the last round with training (either all-SI or temporal-only).


\subsection{Protocols timeline}
The protocol defines the sequence of round types and maximum rate for each round \(r\) which are organized into two phases: an initial warm-up phase done only once at the start of training, and a main phase which is repeated. Fig.~\ref{fig:protocols_timeline} illustrates an example of a protocol timeline.

We will show protocols by listing the sequence of round types and their maximum rates used for each phase. We use abbreviations \(P\), \(A\), \(T\) and \(F\) for Pretrained, All-SI, Temporal-only and Freeze round types respectively, with the first 3 having a subscript indicating the allocated symbols and stage count for the WZQ. For example, \(A_8,3\) means an All-SI round with a WZQ of 3 stages and 8 symbols per stage.

Our main protocol is as follows, which we will refer to as \textbf{Combined Protocol with \(J\) stages} with \(J\) being and int between 1 and 3 depending on out rate budget.
\begin{align*}
    &\text{Warmup phase: } \(P_{8,J}, T_{8,J}, R_{4,J} * 3\) \\
    &\text{Routine phase: } \(T_{2,J}, T_{2,J}, R_{2,J}, F_{2,J} * 6\)
\end{align*}

This particular sequence of using two temporal-only rounds (or pretrained round) is because we can be sure to have two low-distortion reconstructions which can be used to make a high quality SI and training target for the next all-SI round.

There are 2 other protocols we will compare to, which are ablations of the main protocol, one using only temporal-only rounds and the other using only all-SI rounds, called \textbf{Temporal-only Protocol with \(J\) stages} and \textbf{All-SI Protocol with \(J\) stages} respectively.
